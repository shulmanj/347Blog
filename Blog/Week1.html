Inline Style vs CSS in HTML

Although I've done a little bit of HTML work, I still never understood where exactly the style should go. In class today, I was told that style should rarely go in the HTML file, it should go in a CSS file. This had me thinking "What is CSS?".

CSS, or Cascading Style Sheets, is used to style the webpage. The word "Cascading" refers to the fact that a style that is applied to a parent element is also applied to its children. Without CSS, all style would be done in the HTML file and would look something like this:

This is blue text.

or:

This is big text.

CSS allows all of that formatting to be done in a separate file, which in turn will keep the HTML file itself much cleaner and easier to read.

To make the text blue, my code is: 

<blockquote style="color:blue;">This is blue text.</blockquote>

To make the text larger, my code is: 

<blockquote style="font-size:50px;">This is big text.</blockquote>

Ultimately, I can see why using CSS as opposed to inline style in HTML is important. Reading through the HTML of this blog post is a bit challenging at times. Throwing all of the style aspects into its own file will make all of the code look much less cluttered and easier to read.
